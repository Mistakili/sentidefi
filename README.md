## What is SentinelFi?

Autonomous finance has a trust gap: AI agents, wallets, and protocols execute
on-chain actions with no shared safety layer. Existing scanners (GoPlus,
De.Fi, Honeypot.is) are **websites, not protocols** — humans click, agents
can't call, and no verdict is verifiable after the fact.

SentinelFi is built the opposite way: **infrastructure first, UI second**.
The same engine that powers our Copilot ships as a public Trust API and MCP
server, so any agent runtime, wallet, launchpad, or DEX can call it directly.
Verdicts return as signed **Safety Attestations** anchored on **BotChain** —
chain-agnostic core, BotChain as the trust anchor.

## Features

- **MCP-native.** Public `/mcp` endpoint, zero-auth read tools.
- **BotChain-anchored, chain-agnostic core.** Attestations anchor to BotChain's RiskRegistry; new EVMs drop in via `src/lib/chains.ts`.
- **Contract X-ray.** Live on-chain reads: metadata, supply, contract-vs-EOA, bytecode.
- **AI Copilot.** Streaming chat that routes intent — token scan vs wallet lookup vs history.
- **Public risk-scan dataset.** Every verdict is written to a public feed with a tx hash for provenance.
- **Explainable verdicts.** Every score cites the on-chain evidence it used — no black box.
- **Portfolio insight.** Wallet balances, USD values, concentration & stablecoin ratio.

## MCP tools

Point Claude, ChatGPT, Cursor, Codex, or any MCP-aware client at
`https://sentidefi.lovable.app/mcp` and these tools appear automatically:

| Tool | What it does |
|---|---|
| `scan_token` | Live on-chain read for any EVM contract: metadata, supply, contract-vs-EOA, bytecode size. Returns a structured verdict. |
| `get_wallet_portfolio` | Native + tracked ERC20 balances with USD values and total for any address. |
| `list_recent_risk_scans` | The public verdict feed. Optional address filter. Score, level, summary, tx hash. |

All tools are `readOnlyHint: true` and only expose already-public data.

**Example (in any MCP client):**

```
> scan token 0x7f3a...b91e on HSK
→ score: 82  HIGH RISK
  reason: unlocked LP, 64% top holder

> what's in wallet 0xb9C5...Ded66?
→ HSK: 12.4  USDT: 830  total $1,204
```

## Tech stack

- **Framework:** TanStack Start v1 (React 19, Vite 7, SSR on Cloudflare Workers)
- **UI:** Tailwind CSS v4, shadcn/ui, streaming AI SDK chat
- **AI:** Lovable AI Gateway (tool calling, streaming)
- **MCP:** `@lovable.dev/mcp-js` mounted at `/mcp`
- **Data:** Direct JSON-RPC to HSK for balances, metadata, bytecode — no third-party indexer
- **Backend:** Lovable Cloud (Postgres + RLS) for the public risk-scan feed
- **Wallet:** EIP-1193 (MetaMask & compatible), multi-chain aware

## FAQ

**Does SentinelFi ever move funds?**
No. All tools are read-only. Any transaction still requires the user to sign in their wallet.

**Where do risk scores come from?**
From live on-chain data plus LLM reasoning against explicit heuristics (contract-vs-EOA, ERC20 conformance, bytecode size, supply anomalies). Scores are opinions, not guarantees — always DYOR.

**Is the MCP server really public?**
Yes. No auth, no keys, no rate-limit paperwork. All three tools only read already-public data (on-chain reads + the public `risk_scans` feed).

**How do I connect it to Claude / ChatGPT / Cursor?**
Add `https://sentidefi.lovable.app/mcp` as an MCP server in your client. The three tools appear automatically.
